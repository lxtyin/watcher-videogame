# 内容注册与资源组织

## 目标

把“可扩展内容”与“运行时代码”拆开，避免新增工具、地形、角色、召唤物或资源时，需要在很多分散文件里同步改动。

## shared 内容注册

shared 的静态内容集中在 `packages/shared/src/content/`：

- `tools.ts`
  - 工具定义、按钮文案、目标模式、默认参数、是否可调试发放。
- `characters.ts`
  - 角色静态定义与回合开始动作入口。
- `summons.ts`
  - 召唤物静态定义。
- `maps.ts`
  - 地图、模式、出生点与 debug 能力开关。
- `effects.ts`
  - 表现语义类型。
- `boards/defaultBoard.ts`
  - 默认棋盘与符号布局。

这些注册表会驱动 shared 类型与运行时读取。新增内容时，优先改注册表，而不是先改 room 或 client 分支逻辑。

## shared 运行时代码

与内容对应的行为实现应放在 shared 规则层：

- 工具行为：`packages/shared/src/rules/executors/`
- 统一动作入口：`packages/shared/src/actions.ts`
- 回合资源与角色状态：`packages/shared/src/characterRuntime.ts`
- 地形触发：`packages/shared/src/terrain.ts`
- 召唤物触发：`packages/shared/src/summons.ts`
- 回合编排：`packages/shared/src/gameOrchestration.ts`

## server 资源组织

server 不维护玩法内容注册，只维护联机适配：

- `WatcherRoom.ts`
  - 生命周期、消息入口、timer。
- `roomStateMappers.ts`
  - schema -> shared。
- `roomStateMutations.ts`
  - shared -> schema。

因此，新增工具或角色时，server 理论上不需要新增规则文件；最多只需要补字段映射。

## client 内容与资源

客户端的内容与资源分成两层：

### 内容描述

位于 `packages/client/src/game/content/`：

- `actionUi.ts`
  - 工具按钮 token、提示语、强调色。
- `inspectables.ts`
  - 地形、角色、召唤物检查卡内容。
- 其他 UI registry
  - 仅负责界面文案与映射，不负责规则。

### 资源与视觉实现

位于 `packages/client/src/game/assets/`：

- `board/`
- `player/`
- `summons/`
- `presentation/`
- `previews/`

这层存放场景资源、视觉 prefab 与渲染映射。规则仍然来自 shared。

## 新增内容时的推荐入口

### 新增工具

1. 改 `packages/shared/src/content/tools.ts`
2. 改 `packages/shared/src/rules/executors/`
3. 如有前端专属视觉，再改 `packages/client/src/game/content/` 与 `packages/client/src/game/assets/`

### 新增地形

1. 改地图布局或棋盘内容
2. 改 `packages/shared/src/terrain.ts`
3. 如有视觉差异，再改 client 视觉映射

### 新增召唤物

1. 改 `packages/shared/src/content/summons.ts`
2. 改 `packages/shared/src/summons.ts`
3. 改 client summon visual registry

### 新增角色

1. 改 `packages/shared/src/content/characters.ts`
2. 改 `packages/shared/src/characterRuntime.ts`
3. 如有角色专属工具，再走工具扩展路径

## 明确不要做的事

- 不要把新工具规则直接写进 `WatcherRoom.ts`
- 不要把新回合分支写进 `simulation/engine.ts`
- 不要在 `BoardScene.tsx` 里硬编码真实规则判定
- 不要在多处维护同一个内容 id 的重复常量

如果某个扩展必须跨很多层同时改，大概率说明扩展点还不够清晰，应先补 shared 注册与运行时边界。
