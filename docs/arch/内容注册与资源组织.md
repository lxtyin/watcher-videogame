# 内容注册与资源组织

## 目标

把“可扩展内容”与“运行时代码”拆开，保证新增工具、地形、角色、召唤物或表现资源时，能够快速定位唯一入口，并尽量做到一类内容一个目录、一种机制一个模块。

## shared 内容注册

shared 的静态内容主要集中在 `packages/shared/src/content/`，工具定义则随工具模块一起维护：

- `packages/shared/src/tool-modules/`
  - 工具静态定义、默认参数、可用性、交互与执行逻辑
- `characters.ts`
  - 角色静态定义，只负责组织技能
- `summons.ts`
  - 召唤物静态定义；`kind: "object"` 表示普通场上物件，`kind: "creature"` 表示会进入工具命中与位移链路的生物
- `effects.ts`
  - 表现语义类型
- `sounds.ts`
  - 表现层音效 cue 定义，只描述语义 id，不直接绑定 client 文件路径
- `boards/defaultBoard.ts`
  - 默认棋盘与地图描述符；描述符以 tab 分隔格子，单格可用 `地形|特性` 表达初始召唤物等附加内容，例如 `L5` 表示奖励移动 5 的 Lucky，`L:rocket` 表示奖励火箭的 Lucky，`.|p5` 表示空地上初始放置携带移动 5 的骰子猪
- `diceReward.ts`
  - 点数骰、工具骰与随机工具奖励的通用规则 helper，Lucky 和骰子猪共用
- `tileState.ts`
  - 地块状态的复制与判空 helper；需要随地形保存的轻量状态优先放在 `TileDefinition.state`
- `maps.ts`
  - 地图注册表；可通过 `initialSeeds` 固定某张地图的初始移动骰 / 工具骰种子，未设置时仍由房间在开局时生成非固定种子

新增内容时，优先改 shared 的内容模块与注册表，而不是先去 room 或 client 写特判。

## shared 运行时代码

### 工具

- `packages/shared/src/tool-modules/`
  - 一个工具一个主文件
  - 同时管理定义、`isAvailable`、preview、execute 与私有 helper
  - `index.ts` 负责接入工具模块、执行器与工具骰面

### 地形

- `packages/shared/src/terrain-modules/`
  - 一个地形一个文件
  - 当前包含：
    - `floor`
    - `wall`
    - `earth-wall`
    - `boxing-ball`
    - `highwall`
    - `start`
    - `goal`
    - `conveyor`
    - `poison`
    - `pit`
    - `cannon`
    - `lucky`
    - `empty-lucky`
地面移动、飞跃穿越、投射物阻挡与落点合法性属于底层位移和空间规则，由 `movementSystem.ts` 与 `spatial.ts` 维护。
地形自身的通用触发入口包括 `onPassThrough / onStop / onImpact`；是否产生 `impact` 由位移系统或投射物结算 author，地形模块只消费该语义。

### 技能与状态

- `packages/shared/src/skills/`
  - 一个技能一个文件
- `packages/shared/src/modifiers.ts`
  - modifier hook 协议
- `packages/shared/src/playerTags.ts`
  - 玩家标签读写

### 表现音效

- `packages/shared/src/content/sounds.ts`
  - 音效 cue 注册表
- `packages/shared/src/sounds.ts`
  - 音效 cue 查询入口
- `packages/shared/src/rules/actionPresentation.ts`
  - 构造 `sound` presentation 事件

音效何时播放属于 shared 规则的一部分，应由工具、地形或位移系统在结算时写入 `sound` 事件；client 只负责按时间线播放，不反推业务语义。

## server 资源组织

server 不维护玩法内容注册，只维护联机适配层：

- `WatcherRoom.ts`
  - 房间生命周期与消息入口
- `roomStateMappers.ts`
  - schema -> shared
- `roomStateMutations.ts`
  - shared -> schema

因此新增工具、地形、角色或技能时，server 原则上不应出现新的专用规则文件。

## client 内容与资源

### 文案与展示映射

- `packages/client/src/game/content/actionUi.ts`
  - 工具按钮的视觉 token 与强调色
- `packages/client/src/game/content/inspectables.ts`
  - 地形、角色与召唤物信息卡的展示组装

工具、地形、技能的文字说明不应在 client 侧维护第二份。当前统一入口是：

- `getToolTextDescription(tool)`
  - 调用对应工具模块的 `getTextDescription`
  - 展示文本必须由工具模块手写，不能依赖共享参数标签表自动生成
- `getTerrainTextDescription(tile)`
  - 调用对应地形模块的 `getTextDescription`
- `getSkillTextDescription(skillId)`
  - 调用对应技能定义的 `getTextDescription`

client 可以决定这些字段如何排版、用什么素材展示，但不应重写规则含义或数值描述。

### 资源目录

- `packages/client/src/game/assets/board/`
  - 棋盘地形本体、地形装饰、地形 effect 与通用地形 thumbnail
- `packages/client/src/game/assets/tools/shared/`
  - 通用工具预览素材
- `packages/client/src/game/assets/tools/<tool-id>/`
  - 单个工具自己的方向箭头、投射物、effect preview 等资源
- `packages/client/src/game/assets/presentation/`
  - presentation 语义到具体资源的渲染入口，包括通用 effect、link 与 number popup
- `packages/client/src/game/assets/audio/`
  - 音乐与音效素材，按 `bgm/`、`sfx/` 分目录组织
- `packages/client/src/game/assets/summons/`
  - 召唤物资源
- `packages/client/src/game/assets/dice/models/`
  - 点数骰、工具骰与随机骰模型；骰面朝向配置由 `diceRollAnimation.ts` 维护
- `packages/client/src/game/assets/dice/DiceRewardModel.tsx`
  - 奖励骰通用模型入口；Lucky、骰子猪携带物与奖励 effect 都应复用它
- `packages/client/src/game/audio/`
  - 音频运行时、资源映射与播放调度
- `packages/client/src/map-editor/`
  - 地图编辑器页面、棋盘壳层与布局文本导入导出

### 地图编辑器复用规则

- 地图编辑器只维护 `layout` 文本，不引入第二份棋盘数据模型
- 地图编辑器与正式对局共用 `BoardTileVisual`
- 地形库条目来自 shared 默认符号表，而不是 client 自己维护一份独立地形表
- 地图编辑器可通过同一布局描述符放置初始召唤物；更改地形会直接替换格子描述符，从而清除该格初始召唤物
- 地图编辑器与正式对局的地形信息卡共用 `TerrainThumbnailCaptureDeck` / `TerrainThumbnail`
- 自定义地图联机测试复用 `createRoom({ mapId: "custom", customMap })`

## 新增内容时的推荐入口

### 新增工具

1. 在 `packages/shared/src/tool-modules/` 增加工具模块
2. 在工具定义内提供 `interaction / isAvailable / getTextDescription`
3. 在 `packages/shared/src/tool-modules/index.ts` 接入模块、执行器与工具骰面
4. 如需参数，优先使用通用 `movePoints`，工具私有参数直接在工具模块内约定命名
5. 在工具定义内手写前端需要展示的标题、说明与数值文本
6. 如有前端资源，再增加 `packages/client/src/game/assets/tools/<tool-id>/`
7. 如需新的按钮 token 或强调色，再改 `actionUi.ts`

### 新增地形

1. 扩展 `packages/shared/src/content/schema.ts`
2. 在 `packages/shared/src/terrain-modules/` 增加地形文件
3. 在地形模块内提供 `label / accent / getTextDescription`，按需要实现 `onPassThrough / onStop / onImpact`
4. 如需新的底层阻挡、落点或 impact 语义，更新 `movementSystem.ts` 与 `spatial.ts`
5. 如需地形外观或地形特效，再接入 `packages/client/src/game/assets/board/`

### 新增召唤物

1. 修改 `packages/shared/src/content/summons.ts`
2. 选择召唤物 `kind`：`object` 不会被工具当作可移动实体命中，`creature` 会像玩家一样进入工具命中、位移和地形触发链路
3. 修改 `packages/shared/src/summons/` 下对应召唤物模块，并在 `packages/shared/src/summons/index.ts` 接入；生物死亡逻辑使用 `onDeath`
4. 如需地图初始放置，在 tab 分隔的布局描述符中加入召唤物特性，或在地图专属符号表中给符号配置 `initialSummon`
5. 接入 client summon 资源

### 新增角色能力

1. 修改 `packages/shared/src/content/characters.ts`
2. 在 `packages/shared/src/skills/` 增加或复用技能模块
3. 在技能定义内提供 `getTextDescription`
4. 如需要主动能力，再增加工具模块

### 新增音效或音乐

1. 在 `packages/shared/src/content/sounds.ts` 增加或复用音效 cue
2. 在对应的工具、地形、位移或结算逻辑里写入 `sound` 事件
3. 把音频文件放入 `packages/client/src/game/assets/audio/`
4. 在 `packages/client/src/game/assets/audio/audioRegistry.ts` 注册资源、变体与默认音量
5. 如需新的场景级背景音乐切换，再调整 `App.tsx` 与 `GameAudioDirector.tsx`

## 明确不要做的事

- 不要把新工具或新地形规则直接写进 `WatcherRoom.ts`
- 不要在 `simulation/engine.ts` 写第二份玩法实现
- 不要在 `BoardScene.tsx` 里硬编码真实规则判断
- 不要把单个工具或单个地形的专用资源再拆散到大量横切目录中
