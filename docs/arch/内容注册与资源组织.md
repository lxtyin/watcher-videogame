# 内容注册与资源组织

## 目标

把“可扩展内容”和“运行时代码”拆开，避免新增工具、地形、角色、召唤物或视觉资源时，需要在大量分散文件里同时改动。

## shared 内容注册

shared 的静态内容集中在 `packages/shared/src/content/`：

- `tools.ts`
  - 工具静态定义、默认参数、阶段限制、调试开关
- `characters.ts`
  - 角色静态定义，只负责组合 `skillIds`
- `summons.ts`
  - 召唤物静态定义
- `maps.ts`
  - 地图、模式、出生点与 debug 开关
- `effects.ts`
  - 表现语义类型
- `boards/defaultBoard.ts`
  - 默认棋盘布局

新增内容时，优先改注册表，而不是先去 room 或 client 写特判。

## shared 运行时代码

与内容对应的行为实现放在 shared 规则层：

- 工具模块：`packages/shared/src/tool-modules/`
- 技能模块：`packages/shared/src/skills/`
- 动作入口：`packages/shared/src/actions.ts`
- 能力接口：`packages/shared/src/modifiers.ts`
- 玩家标签工具：`packages/shared/src/playerTags.ts`
- 地形触发：`packages/shared/src/terrain.ts`
- 召唤物触发：`packages/shared/src/summons.ts`
- 回合编排：`packages/shared/src/gameOrchestration.ts`

## server 资源组织

server 不维护玩法内容注册，只维护联机适配：

- `WatcherRoom.ts`
  - 生命周期、消息入口、timer
- `roomStateMappers.ts`
  - schema -> shared
- `roomStateMutations.ts`
  - shared -> schema

因此新增工具、角色或技能时，server 原则上不需要新增规则文件；最多只需要补通用字段映射。

## client 内容与资源

### 内容描述

位于 `packages/client/src/game/content/`：

- `actionUi.ts`
  - 工具按钮 token、提示语、强调色
- `inspectables.ts`
  - 地形、角色、召唤物检查卡内容

这一层只负责界面文案与展示映射，不负责真实规则。

### 资源与视觉实现

位于 `packages/client/src/game/assets/`。

当前分层：

- `board/`
  - 棋盘与 tile 资源
- `player/`
  - 玩家与宠物资源
- `tools/shared/`
  - 通用工具 preview 资源与共用类型
- `tools/<tool-id>/`
  - 单个工具自己的方向箭头、投射物、effect preview、召唤物资源
- `presentation/`
  - presentation 语义到具体资产的渲染入口
- `summons/`
  - 召唤物渲染入口

原则：

- 和某个工具强相关的资源，优先收敛到 `assets/tools/<tool-id>/`
- 通用入口只保留 registry 和渲染装配，不再堆大量工具特判

## 新增内容时的推荐入口

### 新增工具

1. 改 `packages/shared/src/content/tools.ts`
2. 改 `packages/shared/src/tool-modules/`
3. 如有 client 专属视觉，再在 `packages/client/src/game/assets/tools/<tool-id>/` 增加资源
4. 如有 UI 文案，再改 `packages/client/src/game/content/actionUi.ts`

### 新增地形

1. 改地图布局或棋盘内容
2. 改 `packages/shared/src/terrain.ts`
3. 如有视觉差异，再改 client 的 board 资源映射

### 新增召唤物

1. 改 `packages/shared/src/content/summons.ts`
2. 改 `packages/shared/src/summons.ts`
3. 改 client summon visual registry

### 新增角色

1. 改 `packages/shared/src/content/characters.ts`
2. 新增或复用 `packages/shared/src/skills/` 中的技能模块
3. 如需要主动能力，再补工具模块
4. 如有前端展示差异，再补 client 内容映射

## 明确不要做的事

- 不要把新工具规则直接写进 `WatcherRoom.ts`
- 不要把新回合分支写进 `simulation/engine.ts`
- 不要在 `BoardScene.tsx` 里硬编码真实规则判定
- 不要在多处重复维护同一份内容 id
- 不要为了单个角色再新增一套并行状态容器
