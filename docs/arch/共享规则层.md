# 共享规则层

## 位置

- `packages/shared/src/constants.ts`
- `packages/shared/src/types.ts`
- `packages/shared/src/board.ts`
- `packages/shared/src/dice.ts`
- `packages/shared/src/tools.ts`
- `packages/shared/src/actions.ts`

## 职责

共享层只保存纯数据和纯逻辑，不依赖浏览器或服务端运行时。

当前职责包括：

- 棋盘尺寸与默认地图配置
- 地块、玩家、回合、事件等共享类型
- 移动骰子与工具骰子的确定性投掷逻辑
- 工具定义注册表
- 移动与工具的统一动作结算入口

## 共享数据模型

当前共享类型的核心设计点有：

- `TurnPhase`
  - 当前支持 `roll` 和 `action` 两个阶段。
- `ToolId`
  - 当前支持 `jump`、`hookshot`、`pivot`、`dash`。
- `ToolDefinition`
  - 集中描述工具标签、说明、目标模式、充能次数和 UI 颜色。
- `ActionResolution`
  - 统一返回角色位置变化、路径、地块变更、受影响玩家和资源消耗。

这样做的目的是让客户端预览和服务端权威判定复用同一份结果结构。

## 骰子与工具注册

`dice.ts` 提供确定性骰子逻辑：

- `rollMovementDie(seed)`
  - 从移动骰面中得到移动点数。
- `rollToolDie(seed)`
  - 从工具骰面中得到本回合工具。

`tools.ts` 提供工具注册表：

- `TOOL_DIE_FACES`
  - 工具骰子可能出现的工具列表。
- `TOOL_DEFINITIONS`
  - 工具元数据的单一来源。
- `isDirectionalTool()`
  - 用于区分定向工具和即时工具。

## 动作结算

`actions.ts` 是当前共享层最核心的入口，包含两个主函数：

- `resolveMovementAction(context)`
  - 消耗一次移动段数，沿选定方向持续前进，直到撞墙、越界、被阻挡或移动点不足。
- `resolveToolAction(context)`
  - 根据 `ToolId` 分发到对应工具执行器，返回统一的 `ActionResolution`。

当前基础工具规则如下：

- `Jump`
  - 朝指定方向跳跃，最多跨越 2 格，优先落到最远可落点。
- `Hookshot`
  - 向前探测 3 格，命中墙体时将自己拉向落点，命中玩家时尝试把对方拉近。
- `Pivot`
  - 立即获得额外一次移动段数。
- `Dash`
  - 立即获得额外 2 点移动点数。

## 扩展方式

当前工具系统按“注册定义 + 注册执行器”的方式扩展，避免把判断逻辑散落在房间和客户端中。

新增工具时，通常只需要同步修改这些位置：

1. 在 `types.ts` 扩展 `ToolId`。
2. 在 `tools.ts` 增加 `TOOL_DEFINITIONS` 条目。
3. 在 `tools.ts` 或配置层决定是否加入 `TOOL_DIE_FACES`。
4. 在 `actions.ts` 增加工具执行器并注册到 `TOOL_EXECUTORS`。

如果未来有新地形或更复杂结算，也可以继续扩展：

- `TileType`
- `TileMutation`
- `AffectedPlayerMove`
- `ActionResolution`

## 设计意图

- 客户端预览和服务端权威校验使用同一份共享规则，减少判定漂移。
- 工具行为集中在共享层，降低前后端重复逻辑。
- 返回结构保持简洁统一，方便继续叠加击退、传送、陷阱、区域效果等能力。
