# 共享规则层

## 目标

`packages/shared` 是 Watcher 的唯一规则权威层。
只要某个机制会影响真实对局结果，它就必须先在这里实现，再由 server 和 client 复用。

## 当前模块划分

### 回合编排

- `packages/shared/src/gameOrchestration.ts`
  - 统一处理高层命令与 `turn-start -> turn-action -> turn-end` 阶段推进
  - 负责回合开始与结束时的停留结算、事件日志、presentation sequence 与竞速结算

### 动作结算

- `packages/shared/src/actions.ts`
  - 统一工具动作入口
- `packages/shared/src/rules/actionDraft.ts`
  - 定义 `ToolActionDraft`，作为 shared 主链的唯一结算草稿
- `packages/shared/src/rules/movementSystem.ts`
  - 统一底层位移结算；所有真实位移都必须经过这里
- `packages/shared/src/rules/previewDescriptor.ts`
  - 统一构造 `PreviewDescriptor`
- `packages/shared/src/rules/actionPresentation.ts`
  - 统一构造 `ActionPresentation`
- `packages/shared/src/rules/actionResolution.ts`
  - 基于 tile / summon mutation 追加 `state_transition`

### 能力系统

- `packages/shared/src/tool-modules/`
  - 按工具拆分的定义、preview 与 execute
- `packages/shared/src/toolInteraction.ts`
  - 工具交互协议与输入结构
- `packages/shared/src/skills/`
  - 按技能拆分的 `Skill -> Modifier`
- `packages/shared/src/modifiers.ts`
  - modifier hook 定义
- `packages/shared/src/playerTags.ts`
  - `Player.tags` 读写辅助

### 地形与召唤物

- `packages/shared/src/terrain-modules/`
  - 每个地形一个文件
  - 地形只能通过接收并修改 draft 的方式干预主流程
- `packages/shared/src/terrain.ts`
  - 地形分发入口
- `packages/shared/src/terrain-modules/traversal.ts`
  - 通用穿越规则
  - 统一定义“是否阻挡地面移动 / 飞跃穿越 / 投射物”
- `packages/shared/src/summons.ts`
  - 召唤物触发入口

### 内容注册

- `packages/shared/src/content/tools.ts`
- `packages/shared/src/content/characters.ts`
- `packages/shared/src/content/summons.ts`
- `packages/shared/src/content/effects.ts`
- `packages/shared/src/content/boards/defaultBoard.ts`

## 地形框架

当前地形系统已经收口成“每种地形一个模块”的模型。

- `floor`
  - 空白地块
- `wall`
  - 阻挡地面移动与投射物
- `earth-wall`
  - 阻挡投射物；被撞碎时通过 presentation 播放碎裂特效
- `highwall`
  - 阻挡地面移动、飞跃穿越与投射物
- `start`
  - 出生点
- `goal`
  - 终点
- `conveyor`
  - 停留时沿朝向移动
- `poison`
  - 停留时触发，把玩家送回出生点
  - 使用玩家 `motion(style = "fall_side")` 表达中毒侧倒
- `pit`
  - 经过时触发，把玩家送回出生点并中断本次移动
  - 使用玩家 `motion(style = "spin_drop")` 表达旋转坠落
- `cannon`
  - 停留时触发，直接复用火箭核心结算
- `lucky`
  - 停留时给予工具，并把地形从 `lucky` 切换为 `emptyLucky`
  - 通过 `reaction + state_transition` 表达领取动画与显隐切换
- `emptyLucky`
  - 空幸运方块
  - 在下一位玩家回合开始时统一恢复为 `lucky`

## Draft 规则

shared 主链已经统一成“自顶向下传递 draft”的结算模式。

- 上层负责决定调用顺序
- 下层 resolver、tool executor、terrain trigger、summon trigger 只负责读取并修改 draft
- 不再允许返回一组 patch 交给上层二次合并

这意味着：

- `Tool.execute()` 应直接修改 `ToolActionDraft`
- 地形与召唤物触发器应直接修改同一个 draft
- 嵌套触发产生的 presentation 应直接追加到同一个 draft 中

## 关键边界

### 回合边界

以下逻辑只允许写在 `packages/shared/src/gameOrchestration.ts`：

- 阶段推进
- 回合开始与结束的停留结算
- finish 动画等待与竞速结算

以下文件不得再编写回合逻辑：

- `packages/server/src/rooms/WatcherRoom.ts`
- `packages/shared/src/simulation/engine.ts`

### 位移边界

所有真实位移都必须经过 `packages/shared/src/rules/movementSystem.ts`。
任何 Tool、Modifier、Terrain 或 Summon 都不应直接修改玩家位置。

### 表现边界

shared 只负责语义层：

- 最终状态变化
- `PreviewDescriptor`
- `ActionPresentation`
- 事件日志

shared 不负责 mesh、材质、shader 或客户端播放细节。

### 地形边界

地形模块只允许依赖通用 draft、通用 traversal 规则与通用 helper。

未来 AI 不应：

- 在 `gameOrchestration.ts` 中为单个地形加专用变量
- 在 `movementSystem.ts` 中为单个地形加散点特判
- 在 room 或 simulator 中重写地形结算

## 扩展入口

### 新增工具

1. 在 `packages/shared/src/content/tools.ts` 注册静态定义
2. 在 `packages/shared/src/tool-modules/` 增加模块
3. 通过 draft 完成 preview 与 execute

### 新增地形

1. 在 `packages/shared/src/content/schema.ts` 扩展 tile 类型
2. 在 `packages/shared/src/terrain-modules/` 增加单文件模块
3. 如需通用阻挡规则，优先扩展 `traversal.ts`
4. 如需前端表现，再接入 client 的 board 资源与 presentation 资源

### 新增召唤物

1. 在 `packages/shared/src/content/summons.ts` 注册定义
2. 在 `packages/shared/src/summons.ts` 接入触发逻辑

## 对未来 AI 的要求

修改 shared 规则前，先检查：

1. 这是否影响真实结算结果
2. 如果影响，是否应该进入 shared 主链
3. 新逻辑是否在修改 draft，而不是返回 patch
4. 是否把地形、工具、技能的专用规则收进各自模块，而不是写进 orchestrator、room 或 scene
