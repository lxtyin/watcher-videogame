# 架构总览

## 三层结构

- `packages/shared`
  - 唯一规则权威层
  - 负责内容注册、回合编排、动作结算、移动系统、地形、召唤物、能力系统与表现语义
- `packages/server`
  - Colyseus authoritative room 适配层
  - 负责房间生命周期、消息入口、schema 同步、重连与大厅流程
- `packages/client`
  - React + Zustand + React Three Fiber 表现层
  - 负责输入、预览、播放、HUD 与房间界面

## 主数据流

1. client 发送高层命令，例如 `rollDice`、`useTool`、`endTurn`。
2. `WatcherRoom` 把 schema 映射为 shared snapshot 与 runtime。
3. room 调用 shared orchestration。
4. shared 完成回合推进、能力触发、动作结算与表现语义生成。
5. room 把 shared 结果回写到 schema。
6. client 订阅 schema 并反序列化成本地 snapshot。
7. client 本地基于 shared 规则计算 preview。
8. client 把 `latestPresentation` 交给 `PlaybackEngine` 播放。
9. `BoardScene` 只消费 preview 与 playback 结果做渲染。

## 当前最重要的边界

### 回合边界

- 只有 `packages/shared/src/gameOrchestration.ts` 可以编写回合逻辑。
- `WatcherRoom.ts` 与 `packages/shared/src/simulation/engine.ts` 只允许做适配与调度。

### 规则与表现边界

- shared 只产出语义结果，不关心具体 mesh。
- client 只负责输入、播放与渲染，不重写真实规则。

### 能力边界

- 主动能力统一建模为 `Tool`
- 被动能力统一建模为 `Skill -> Modifier`
- 跨回合状态统一落在 `Player.tags + Player.modifiers`
