# 架构总览

## 目标

当前实现遵循需求文档中的三层拆分：

- `packages/shared`
  - 放置前后端共用的类型、常量、棋盘定义、骰子逻辑与动作结算函数。
- `packages/server`
  - 使用 Colyseus 维护权威房间状态，验证掷骰、移动、使用工具和结束回合。
- `packages/client`
  - 使用 React、Zustand 和 React Three Fiber 呈现 3D 原型，并把玩家意图发送给服务端。

## 当前运行链路

1. 客户端通过 `colyseus.js` 连接 `watcher_room`。
2. 服务端在玩家加入时创建棋子状态，并把首个玩家置为当前行动者。
3. 当前玩家在 `roll` 阶段发送 `rollDice`，服务端使用共享层骰子逻辑得到移动点数和本回合工具。
4. 服务端把投骰结果写入玩家资源和 `turnInfo`，回合进入 `action` 阶段。
5. 客户端在 HUD 中选择 `Move` 或某个工具，并在 3D 棋盘内通过拖动棋子或点选同轴地块选择方向。
6. 服务端调用共享层动作结算函数，产出新的角色位置、地块变化和受影响玩家位置。
7. Colyseus 将房间状态同步回客户端，客户端据此刷新 3D 场景、HUD 和事件日志。

## 当前已实现内容

- 7x7 原型棋盘
- `floor`、`wall`、`earthWall` 三种基础地形
- 双阶段回合流：`roll` -> `action`
- 双骰原型：移动骰子 + 工具骰子
- 基础工具原型：`Jump`、`Hookshot`、`Pivot`、`Dash`
- 基于共享规则的客户端路径预览
- 3D 场景内执行动作：拖动棋子定向，点选同轴地块触发动作
- 占位素材：cube 地块、椭球棋子、基础高亮与轮廓标记
- `window.render_game_to_text` 与 `window.advanceTime` 自动化桥接
