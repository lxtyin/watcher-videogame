# 表现层原型

## 目的

本文档定义当前 client 表现层的唯一权威模型。未来 AI 如需修改预览、动画、播放或工具表现，应以本文为准，不应恢复旧的散点式播放逻辑。

## 三个核心概念

### PreviewDescriptor

`PreviewDescriptor` 是 shared 规则层给 client 的预览描述结构。它只表达语义，不表达 mesh、材质或时间曲线。

当前字段：

- `actorPath`
  - 当前玩家的预计移动路径
- `selectionTiles`
  - 选择地块时可选择的范围
- `effectTiles`
  - 工具预计作用到的地块范围
- `playerTargets`
  - 所有受影响玩家的预计落点
- `valid`
  - 当前输入是否可以真正执行

约束：

- preview 在 client 本地基于 shared 规则即时计算
- preview 不依赖额外网络请求
- shared 只负责给出语义字段，如何绘制交给 client
- 不允许再为单个工具增加 `wallGhostPositions`、`summonPreviews` 这类专用 preview 字段

### ActionPresentation

`ActionPresentation` 是 authoritative 动作已经发生后的语义事件序列。当前只允许三类事件：

- `motion`
  - 玩家或投射物运动
- `reaction`
  - 纯表现反应，不改变逻辑状态
- `state_transition`
  - 真正切换显示状态

约束：

- shared 只描述事件语义和时间
- client 不应再自行发明第二套动画事件类型
- 需要延后切图、延后隐藏、延后切可见性时，必须表达成 `state_transition`
- 需要纯视觉反馈时，必须表达成 `reaction`

### PlaybackEngine

`PlaybackEngine` 是 client 唯一的播放引擎，负责把 `ActionPresentation` 采样成“当前这一帧应该显示什么”。

当前职责：

- 推进 `latestPresentation` 的本地播放时间
- 采样玩家瞬态位置、投射物、reaction 与显示状态
- 向场景层输出当下时刻的瞬态结果

硬约束：

- `BoardScene` 不得再实现第二套播放算法
- 不得恢复分散的显态回滚逻辑
- 播放期间 client 不允许继续发起新输入

## 当前实现分层

### shared

- `packages/shared/src/actions.ts`
  - 统一动作解析入口
- `packages/shared/src/rules/previewDescriptor.ts`
  - 统一构造 `PreviewDescriptor`
- `packages/shared/src/rules/actionPresentation.ts`
  - 构造 `ActionPresentation`
- `packages/shared/src/rules/actionResolution.ts`
  - 把真实结算、preview 和 presentation 收口到同一结果对象

### client

- `packages/client/src/game/interaction/toolInteraction.ts`
  - 本地交互会话与输入采集
- `packages/client/src/game/utils/boardMath.ts`
  - 基于 shared 规则做本地 preview 调用
- `packages/client/src/game/animation/playbackEngine.ts`
  - 唯一播放引擎
- `packages/client/src/game/interaction/previewState.ts`
  - 把 `PreviewDescriptor` 转成场景可消费的预览状态
- `packages/client/src/game/components/BoardScene.tsx`
  - 只负责组装 renderer

## 预览链路

1. client 持有最近一次 authoritative snapshot。
2. 玩家在本地进行工具交互，形成当前输入状态。
3. client 本地调用 shared preview 入口，得到 `PreviewDescriptor`。
4. `previewState.ts` 把语义字段拆成场景状态：
   - `selectionTiles -> selectionKeys`
   - `effectTiles -> ToolEffectPreview`
   - `playerTargets -> landingRings`
5. `BoardScene` 消费这些结果并渲染。

当前视觉映射约定：

- `selectionTiles`
  - 统一走地块可选范围高亮
- `effectTiles`
  - 走工具专属 effect preview 资产
- `playerTargets`
  - 所有玩家都显示落点 ring，颜色使用各自玩家颜色

## 播放链路

1. server 只同步 `latestPresentation`。
2. client 把它放入本地播放队列。
3. `PlaybackEngine` 基于本地时钟推进播放。
4. `BoardScene` 只消费播放结果，不再自己推导状态回滚或动画顺序。

## client 资源组织

工具相关资源按工具目录聚合：

- `packages/client/src/game/assets/tools/<tool-id>/`
  - 某个工具自己的方向箭头、投射物、effect preview、召唤物资源
- `packages/client/src/game/assets/tools/shared/`
  - 通用预览资源与工具资源共用类型

当前通用渲染入口：

- `BoardTileVisual`
  - 渲染 `selectionTiles`
- `ToolEffectPreview`
  - 渲染 `effectTiles`
- `SceneDirectionArrows`
  - 渲染方向型工具指示
- `ProjectileVisual`
  - 渲染 presentation 中的投射物
- `EffectVisual`
  - 渲染 presentation 中的纯表现 effect
- `SummonVisual`
  - 渲染召唤物

## 对未来 AI 的明确约束

未经批准，不应：

- 在 shared 里写具体 mesh、材质或 shader 细节
- 在 client 里重写真实规则
- 在 `BoardScene` 里新增工具专用 preview 字段
- 在 `BoardScene` 里新增第二套播放或回滚逻辑
- 绕开 `PlaybackEngine` 直接拼时间线状态

如果现有模型无法表达新的表现需求，正确做法是先更新本文档，再改代码。
