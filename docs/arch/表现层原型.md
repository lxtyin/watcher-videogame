# 表现层原型

## 目标

本文定义当前 `PreviewDescriptor + ActionPresentation + PlaybackEngine` 的权威模型。
未来 AI 如需修改预览、动画或地形显示规则，应先遵守本文边界，而不是重新在 client 散写特判。

## 三个核心概念

### PreviewDescriptor

`PreviewDescriptor` 是 shared 规则层给 client 的预览语义结构。
它不描述具体 mesh，只描述“这一刻应该展示哪些语义信息”。

当前关键字段：

- `actorPath`
  - 当前玩家的预计移动路径
- `selectionTiles`
  - 选择地块时可选的范围
- `effectTiles`
  - 当前工具预计会作用到的地块
- `highlightTiles`
  - 当前工具预计会触发地形强调表现的地块
  - shared 只给出地块坐标，具体显示方式由 client 按当前地形类型决定
  - 由工具或地形在结算过程中主动写入 preview draft，不从其它字段做终结阶段推导
- `playerTargets`
  - 所有受影响玩家的预计落点
- `valid`
  - 当前输入是否合法

当前约束：

- preview 在 client 本地基于 shared 规则即时计算
- `selectionTiles` 是地块选择范围的唯一权威语义
- `effectTiles` 只表达工具效果，不表达“是否可选”
- `highlightTiles` 只表达需要强调的地块，不表达具体 mesh、材质或动画
- 所有影响玩家位置的工具，都应通过 `playerTargets` 给出所有玩家落点

### ActionPresentation

`ActionPresentation` 是 authoritative 动作发生后的语义事件序列。
当前允许四类事件：

- `motion`
  - 玩家、生物召唤物或投射物运动
- `reaction`
  - 纯表现反馈，不改逻辑状态
- `state_transition`
  - 真正切换显示状态
- `sound`
  - 在指定时刻触发一个音效 cue，可附带 anchor 供未来空间音效或声像使用
- `PresentationAnchor`
  - 当前支持玩家、召唤物与静态位置；链条、音效等表现应通过 anchor 追踪移动中的主体

当前重要约束：

- 需要纯表现反馈时使用 `reaction`
- 需要延后切状态时使用 `state_transition`
- 需要播放音效时使用 `sound`
- 不允许在 client 侧发明第二套并行动画语义
- presentation 的 motion 事件在 shared 生成与追加时按 `startMs` 稳定排序；非 motion 事件保留原槽位和相对顺序
- `sound` 是触发型事件，不承担动作时长，当前约定 `durationMs = 0`
- `TileMutation` 可以携带可选 `presentationStartMs`，用于表达“状态变化已决定，但显示切换要对齐到某个语义时刻”

当前已经接入的地形相关表现：

- `motion(style = "impact_recoil")`
  - 平移撞上 solid tile 后，角色继续顶到墙边再自然弹回
- `motion(style = "fall_side")`
  - poison 触发时的侧倒
- `motion(style = "spin_drop")`
  - pit 触发时的旋转下坠
- `reaction.effect(kind = "boxing_ball_hit")`
  - 拳击球被任意 impact 命中时的摇摆表现
- `reaction.number_popup`
  - 拳击球被当前回合玩家撞击时的数值弹字
- `reaction.effect(kind = "lucky_claim")`
  - lucky 触发时的抬升、虚化、消失
- `reaction.effect(kind = "earth_wall_break")`
  - 土墙破碎时的碎裂特效
- `reaction.effect(kind = "tower_impact")`
  - 塔被撞击时的冲击与碎片特效；若同时发生耐久变化或击碎，tile 的 `state_transition` 应与撞击接触帧对齐

### PlaybackEngine

`PlaybackEngine` 是 client 唯一的播放引擎，负责把 `ActionPresentation` 采样成当前帧的瞬态结果。

它负责：

- 推进 `latestPresentation`
- 计算玩家、投射物与 reaction 的瞬时显示状态
- 计算玩家、生物召唤物、投射物与 reaction 的瞬时显示状态
- 输出当前时刻的场景渲染输入

它不负责：

- 真实规则判断
- 新增工具专用状态机
- 重新解释地形逻辑
- 声音播放调度；音频由独立的 `GameAudioDirector` 消费 `sound` 事件

## 当前实现分层

### shared

- `packages/shared/src/rules/previewDescriptor.ts`
  - 构造 `PreviewDescriptor`
- `packages/shared/src/rules/actionPresentation.ts`
  - 构造 `ActionPresentation`
- `packages/shared/src/rules/actionResolution.ts`
  - 基于 mutation 追加 `state_transition`
- `packages/shared/src/actions.ts`
  - 把真实结算、preview 与 presentation 收口到同一条链路

### client

- `packages/client/src/game/interaction/toolInteraction.ts`
  - 本地交互会话
- `packages/client/src/game/interaction/previewState.ts`
  - 把 `PreviewDescriptor` 转成场景可消费的预览状态
- `packages/client/src/game/animation/playbackEngine.ts`
  - 唯一播放引擎
- `packages/client/src/game/audio/GameAudioDirector.tsx`
  - 读取 shared `sound` 事件并调度音频播放
- `packages/client/src/game/assets/audio/audioRegistry.ts`
  - 维护音频 cue、变体资源与默认音量映射
- `packages/client/src/game/assets/presentation/NumberPopupReactionVisual.tsx`
  - 渲染通用数值弹字 reaction
- `packages/client/src/game/assets/player/PlayerStatusVisuals.tsx`
  - 根据 `Player.modifiers + Player.tags` 渲染持续状态表现，如眩晕与束缚
- `packages/client/src/game/components/BoardScene.tsx`
  - 只负责组装 renderer

## 地形显示规则

### Lucky

`lucky` 现在通过真实 tile mutation 表达“被拿走”。
当前规则是：

- 停留触发时由 shared 发送 `lucky_claim` effect
- 同一时刻把地形从 `lucky` 切换到 `emptyLucky`
- client 只根据 `tile.type` 与 `state_transition` 显示 Lucky 方块是否存在
- 每位玩家回合开始时，shared 把所有 `emptyLucky` 恢复成 `lucky`

也就是说：

- lucky 的可见性属于真实地形状态，而不是 client 显示特判
- 未来 AI 不要再通过当前玩家 `turnFlags` 控制 Lucky 显示

### Highwall / Poison / Pit

- `highwall`
  - 通过独立 tile asset 表达高墙本体与上方栅栏
- `poison`
  - 使用毒气装饰层
- `pit`
  - 保持地块本体不变，动画通过玩家 motion 完成

## 预览映射规则

当前 client 约定：

- `selectionTiles`
  - 统一用可选范围高亮展示
- `effectTiles`
  - 统一交给工具 effect preview 渲染
- `playerTargets`
  - 统一绘制为玩家颜色 ring
- `highlightTiles`
  - 由棋盘地形资源根据地形类型替换或补充预览态：
    - 顺向经过传送带加速时，传送带元素改为绿色
    - 即将击碎土墙时，土墙显示为虚化倒塌态
    - 即将触发毒气或坑洞时，地块上方显示危险提示

未来 AI 不应重新引入：

- `wallGhostPositions`
- `summonPreviews`
- 任何只服务于单个工具的 preview 专用字段

## 资源组织

### 棋盘资源

- `packages/client/src/game/assets/board/`
  - 地形 tile、本体装饰与地形 effect

当前新增地形相关资源包括：

- `BoxingBallTileAsset`
- `PoisonTileAsset`
- `HighwallTileAsset`
- `LuckyClaimEffectAsset`

### 工具资源

- `packages/client/src/game/assets/tools/<tool-id>/`
  - 单个工具自己的投射物、方向箭头、effect preview 等资源

## 对未来 AI 的要求

未经批准，不应：

- 在 `BoardScene.tsx` 内重写预览或播放逻辑
- 在 client 侧绕开 `PlaybackEngine` 重新解释视觉播放，或绕过 `sound` 事件直接硬编码业务音效
- 为单个地形或工具新增 preview 专用状态字段
- 在 shared 里写具体 mesh 与材质细节
