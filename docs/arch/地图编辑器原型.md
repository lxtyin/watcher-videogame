# 地图编辑器原型

## 目标

地图编辑器是一个独立的 web 页面，入口为 `/mapeditor`。它的职责不是重新实现一套棋盘和地形表现，而是复用现有 shared 布局格式、client 单格渲染和 server 建房链路，让自定义地图能够直接进入联机测试。

## 页面入口

- `packages/client/src/main.tsx`
  - 根据 `window.location.pathname === "/mapeditor"` 切换到 `MapEditorApp`
- `packages/client/src/map-editor/MapEditorApp.tsx`
  - 编辑器页面壳层

## 当前状态模型

编辑器当前维护一份轻量文档状态：

- `mapLabel`
- `mode`
- `allowDebugTools`
- `layout`

其中 `layout` 直接沿用 shared 的布局字符串数组格式，不引入第二种棋盘数据结构。每行以 tab 分隔格子，单格描述符可写成 `地形|特性`，例如 `.|p5` 表示空地上初始放置携带移动 5 的骰子猪。编辑器展示和联机建房都从这份 `layout` 重新构造 `BoardDefinition` 与初始召唤物。

## shared 复用入口

- `packages/shared/src/board.ts`
  - `createBoardDefinitionFromLayout()`
  - `createInitialSummonsFromLayout()`
  - `resizeBoardLayout()`
  - `setLayoutCellDescriptor()`
  - `getBoardSpawnPosition()`
- `packages/shared/src/content/boards/defaultBoard.ts`
  - `DEFAULT_BOARD_SYMBOLS`

编辑器只允许摆放当前已有地形与已接入的初始召唤物，因此默认符号表和 shared 布局特性解析是唯一权威来源。

## client 渲染复用

- `packages/client/src/game/assets/board/BoardTileVisual.tsx`
  - 仍然是地图编辑器和正式对局共享的单格渲染入口
- `packages/client/src/map-editor/MapEditorCanvas.tsx`
  - 复用现有相机参数和 OrbitControls 方案
- `packages/client/src/map-editor/MapEditorScene.tsx`
  - 只负责 hover、ghost、初始召唤物预览和拖拽涂抹
- `packages/client/src/game/assets/board/TerrainThumbnailCaptureDeck.tsx`
  - 用小型 R3F Canvas 复用 `BoardTileVisual` 生成地形缩略图
- `packages/client/src/game/assets/board/TerrainThumbnail.tsx`
  - 展示缓存后的地形缩略图，地图编辑器和正式对局的信息卡共用

未来 AI 不应为地图编辑器另写一套 tile mesh、地形装饰或材质规则。任何单格外观调整都应先修改 `BoardTileVisual` 及其依赖资源，再让编辑器和正式对局共同受益。

## 导入导出

- `packages/client/src/map-editor/layoutFormat.ts`

导入导出文本仍然保持 shared 布局风格：

- 导出 `CUSTOM_BOARD_LAYOUT`
- 约定 `CUSTOM_BOARD_SYMBOLS = DEFAULT_BOARD_SYMBOLS`
- `mapLabel / mode / allowDebugTools` 通过顶部注释保留
- 布局行使用 tab 分隔格子；`R` 键切换当前选中项形态，包括方向、阵营或骰子猪携带内容

因此编辑器导入导出的是“默认符号表上的布局文本”，而不是新的 JSON 协议。

## 联机测试入口

- `packages/client/src/game/network/useWatcherConnection.ts`
  - `createRoom()` 现在支持 `{ mapId: "custom", customMap }`
- `packages/server/src/rooms/WatcherRoom.ts`
  - `onCreate()` 接收 `customMap`
  - 自定义地图通过 `createBoardDefinitionFromLayout()` 建立棋盘
  - 自定义地图的出生点通过 `getBoardSpawnPosition(board, mode, playerIndex)` 从 `start` 地块推导

room 仍然使用原有的 `WatcherRoom + shared orchestration` 主链，不单独为地图编辑器维护一套试玩模式。

## 对未来 AI 的要求

- 不要为地图编辑器新增第二份棋盘数据模型
- 不要为地图编辑器重写一套地形渲染规则
- 不要为自定义地图开房再做第二个 room 类型
- 如果要扩展地形库，先扩 shared 的 `DEFAULT_BOARD_SYMBOLS`、`terrain-modules` 文本描述和单格资源，再接编辑器目录
