# 交互层统一原型

## 目的

本文档记录当前已经落地的工具交互模型。未来 AI 如需扩展工具交互，应以这里为准，而不是恢复旧的 `targetMode`、`tileTargeting`、`AimState` 或平铺 payload。

## 分层

### shared

- 定义 `Tool.interaction`
- 定义 `ToolSelectionRecord`
- 基于 `input` 计算 `PreviewDescriptor`
- 基于 `input` 执行 authoritative `ActionResolution`

### client

- 维护本地 `InteractionSession`
- 解释鼠标、键盘和弹窗输入
- 把当前输入状态转换为 `ToolSelectionRecord`
- 调用 shared preview
- 在最终确认时发送 `useTool`

## 输入契约

每个 `Tool` 在 shared 中都必须完整定义：

- `definition`
- `interaction`
- `isAvailable`
- `execute`

约束：

- 合法范围由工具自己的规则逻辑决定
- preview 内容由工具自己的规则逻辑决定
- client 不根据交互模式反推规则合法范围

当前 `useTool` 的权威协议：

```ts
interface UseToolCommandPayload {
  toolInstanceId: string;
  input: ToolSelectionRecord;
}
```

`ToolSelectionRecord` 是命名 slot 到选择值的映射。当前基础选择值只有三种：

- `direction`
- `tile`
- `choice`

## InteractionSession

client 统一维护本地交互会话：

```ts
interface InteractionSession {
  toolId: ToolId;
  toolInstanceId: string;
  stageIndex: number;
  committed: ToolSelectionRecord;
  draft: ToolSelectionRecord;
}
```

语义：

- `committed`
  - 已完成阶段的输入
- `draft`
  - 当前阶段的临时输入
- shared preview
  - 读取 `committed + draft`
- shared execute
  - 读取最终确认后的 `input`

## 当前基础阶段

- `drag-direction-release`
  - 拖动选择方向，松开立即确认
- `drag-tile-release`
  - 拖动选择地块，松开立即确认
- `drag-axis-tile-release`
  - 在一次拖动中同时确定方向和目标格
- `modal-choice`
  - 在弹窗中做离散选择
- `sequence([...])`
  - 按顺序组合多个阶段

## 当前工具映射

- `movement / jump / hookshot / basketball / rocket / awmShoot`
  - `drag-direction-release`
- `buildWall / teleport`
  - `drag-tile-release`
- `brake`
  - `drag-axis-tile-release`
- `bombThrow`
  - `sequence([drag-tile-release, drag-direction-release])`
- `balance`
  - `modal-choice`

## 对未来 AI 的约束

未经批准，不应：

- 恢复 `targetMode` 或 `tileTargeting`
- 在 client 恢复 `AimState`
- 向 `UseToolCommandPayload` 恢复平铺的 `direction / targetPosition / choiceId`
- 为单个工具在 room、simulator 或 client store 中新增专用输入字段
- 让 client 根据交互模式推导规则合法范围

如果需要新的复杂交互，正确做法是：

1. 在 shared 扩展通用 `Tool.interaction` 阶段模型。
2. 在 client 扩展 `InteractionSession` 的解释方式。
3. 仍然通过 `ToolSelectionRecord` 传递最终输入。
