# 前后端联机原型

## 目标

联机层的职责是把 shared 规则接到真实房间与 client 表现上，而不是重新实现游戏规则。

## 服务端

### 关键文件

- `packages/server/src/rooms/WatcherRoom.ts`
- `packages/server/src/rooms/roomStateMappers.ts`
- `packages/server/src/rooms/roomStateMutations.ts`
- `packages/server/src/schema/WatcherState.ts`

### 当前职责

- 创建房间与地图状态
- 支持内建地图与 `customMap` 自定义地图开房
- 处理加入、离开、重连、准备、开局、回房、踢人
- 接收客户端高层命令
- 把 schema 映射到 shared snapshot 与 runtime
- 调用 shared orchestration
- 把 shared 结果回写到 schema
- 按 presentation 时长调度 `advanceTurn()`

### 明确不负责

`WatcherRoom.ts` 不负责：

- 选择下一位玩家
- 编写 turn flow
- race 排名推进
- 回合停留结算

这些逻辑都已经收口到 `packages/shared/src/gameOrchestration.ts`。

## 客户端

### 关键文件

- `packages/client/src/game/network/useWatcherConnection.ts`
- `packages/client/src/game/utils/deserializeRoomState.ts`
- `packages/client/src/game/interaction/toolInteraction.ts`
- `packages/client/src/game/utils/boardMath.ts`
- `packages/client/src/game/animation/playbackEngine.ts`
- `packages/client/src/game/components/BoardScene.tsx`

### 当前职责

- 建立 Colyseus 连接
- 在 `/mapeditor` 复用同一个 `createRoom()` 入口创建自定义地图房间
- 把 schema 反序列化为前端 snapshot
- 提供本地输入、preview 与 HUD
- 在本地基于 shared 规则计算 `PreviewDescriptor`
- 消费 `latestPresentation` 并交给 `PlaybackEngine` 播放
- 在 UI 层阻止明显非法输入，但不承担规则权威

## 自定义地图开房

- `packages/client/src/map-editor/MapEditorApp.tsx`
  - 维护编辑器文档状态，并在测试按钮中调用 `createRoom({ mapId: "custom", customMap })`
- `packages/client/src/game/network/useWatcherConnection.ts`
  - `createRoom()` 同时支持普通 `mapId` 与 `customMap`
- `packages/server/src/rooms/WatcherRoom.ts`
  - `onCreate()` 在有 `customMap` 时直接用 `createBoardDefinitionFromLayout()` 建盘
  - 出生点通过 `getBoardSpawnPosition(board, mode, playerIndex)` 从 `start` 地块推导

自定义地图没有单独的联机协议，也没有第二套房间实现；它只是把创建房间时的棋盘来源从内建 map id 扩展为布局文本。

## 同步链路

1. client 发送高层命令。
2. room 把 schema 映射到 shared snapshot 与 runtime。
3. room 调用 shared orchestration。
4. shared 返回新 snapshot、runtime 与 `latestPresentation`。
5. room 把结果回写到 schema。
6. client 订阅 schema 并反序列化为本地 snapshot。
7. client 本地基于 snapshot 和当前输入计算 preview。
8. client 用 `PlaybackEngine` 播放 `latestPresentation`。

## schema 与 shared snapshot 的关系

schema 是网络同步格式，shared snapshot 是规则格式。两者之间只通过专门映射层转换：

- `roomStateMappers.ts`
  - schema -> shared
- `roomStateMutations.ts`
  - shared -> schema

当前能力相关字段边界：

- schema 同步 `tagsJson + modifiers`
- shared snapshot 同步 `Player.tags + Player.modifiers`
- 角色静态能力仍由 shared 内容注册与技能系统推导

## 对未来 AI 的要求

- 想改消息协议、重连、房间阶段或 timer，可以改 `WatcherRoom.ts`
- 想改 schema 与 snapshot 的字段映射，可以改 mapper / mutation
- 想改真实结算、换人、终点推进、回合开始逻辑，必须回到 `packages/shared/src/gameOrchestration.ts`
- 想改 preview 语义字段，必须回到 shared `PreviewDescriptor`
- 想改播放语义事件，必须回到 shared `ActionPresentation`
- 想改播放算法，必须收口到 `packages/client/src/game/animation/playbackEngine.ts`
