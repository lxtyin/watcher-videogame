# 前后端联机原型

## 服务端

### 位置

- `packages/server/src/index.ts`
- `packages/server/src/schema/WatcherState.ts`
- `packages/server/src/rooms/WatcherRoom.ts`

### 当前职责

- 创建 `watcher_room`
- 初始化权威棋盘状态
- 处理玩家加入、离开与回合切换
- 校验 `rollDice`、`move`、`useTool`、`endTurn`
- 将共享层动作结算结果写回房间状态
- 维护事件日志和最近回合信息

### 状态同步模型

服务端状态根节点是 `WatcherState`，包含：

- `board`
  - Colyseus `MapSchema`，保存地块状态。
- `players`
  - Colyseus `MapSchema`，保存玩家位置、剩余移动点、剩余移动段数和本回合工具充能。
- `turnInfo`
  - 保存当前玩家、回合阶段、投出的移动点数、剩余资源和本回合工具结果。
- `eventLog`
  - 保存最近几条行为日志。

`WatcherRoom` 的关键流程如下：

1. 玩家加入后分配出生点。
2. 当前玩家进入 `roll` 阶段。
3. 收到 `rollDice` 后生成移动骰与工具骰结果。
4. 收到 `move` 或 `useTool` 后调用共享层结算函数。
5. 将位置变化、地块变化和工具消耗写回房间状态。
6. 收到 `endTurn` 后切换到下一个玩家并重置为 `roll` 阶段。

## 客户端

### 位置

- `packages/client/src/App.tsx`
- `packages/client/src/game/network/useWatcherConnection.ts`
- `packages/client/src/game/state/useGameStore.ts`
- `packages/client/src/game/components/BoardScene.tsx`
- `packages/client/src/game/utils/boardMath.ts`

### 当前职责

- 与 Colyseus 房间建立连接
- 将房间状态反序列化为前端快照
- 保存 HUD 和 3D 交互所需的本地状态
- 发送掷骰、移动、使用工具和结束回合请求
- 在 3D 场景中提供拖动定向、点选定向和路径预览

### 当前交互模型

`App.tsx` 负责 HUD 和全局输入：

- 掷骰与结束回合按钮
- 当前投骰信息展示
- 工具选择区与工具说明
- 键盘兜底输入：`R`、`E`、方向键、`WASD`、`Enter`

`BoardScene.tsx` 负责 3D 棋盘交互：

- 拖动自己的棋子来选择方向
- 点选与自己同轴的地块来直接执行当前动作
- 使用共享层动作结算结果生成路径预览
- 用高亮颜色区分移动与不同工具

### 表现层约束

当前原型仍使用占位资源：

- cube 表示地块和障碍
- 椭球棋子表示玩家
- ring 和 circle 表示当前行动者和底部标记

## 自动化桥接

客户端暴露了两个测试接口：

- `window.render_game_to_text`
  - 输出当前可玩状态的 JSON 文本，供自动化读取。
- `window.advanceTime`
  - 允许测试脚本稳定推进时间。

这两个接口用于 Playwright 烟测和后续回归验证。

## 当前已验证行为

- 客户端可以连接房间并进入回合流程
- 当前玩家可以先掷骰，再执行移动或工具
- 移动会正确消耗移动点数和移动段数
- 工具选择会同步到 HUD 与场景预览
- 客户端路径预览与服务端权威结算复用同一套共享规则
