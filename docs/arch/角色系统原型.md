# 角色系统原型

## 目标

这一层用于承载“角色差异化能力”，并保持下面几条边界：

- 角色不直接改前端 UI 逻辑
- 角色不直接改 Colyseus 房间里的散乱特判
- 角色通过共享层的配置与规则，影响每回合 Tool 列表和执行结果

## 当前结构

- `packages/shared/src/characters.ts`
  - 角色注册表
  - 角色回合开始赠送 Tool
  - 角色主动技能 Tool 注入
  - 角色对已有 Tool 的被动变换
- `packages/shared/src/summons.ts`
  - 召唤物共享数据结构
  - 召唤物的“经过时”触发规则
- `packages/server/src/rooms/WatcherRoom.ts`
  - 同步 `characterId`
  - 同步 `summons`
  - 处理 `setCharacter`
  - 在掷骰时构建角色专属回合 Tool 列表
- `packages/client/src/App.tsx`
  - 左栏展示角色信息、被动说明、主动技能按钮、切换角色按钮
- `packages/client/src/game/components/BoardScene.tsx`
  - 渲染钱包召唤物
  - 渲染钱包放置预览

## 角色定义方式

每个角色当前都由一份声明式配置组成，主要包含：

- `turnStartGrants`
  - 每回合开始时额外加入的 Tool
- `activeSkillLoadout`
  - 作为主动技能注入到本回合 Tool 列表中的 Tool
- `toolTransforms`
  - 对已有 Tool 做被动替换或参数映射

这意味着角色能力本质上仍然是对 Tool 系统的扩展，而不是平行于 Tool 的另一套动作框架。

## 当前示例角色

### Late

- 被动：所有 `Movement` 变为 `Brake`
- 实现方式：
  - 在 `toolTransforms` 中把 `movement -> brake`
  - 把 `movePoints` 映射到 `brakeRange`

### Ehh

- 被动：每回合额外获得一个 `Basketball`
- 实现方式：
  - 在 `turnStartGrants` 中追加 `basketball`

### Leader

- 主动：获得 `deployWallet`
- `deployWallet`
  - 在 5x5 范围内选择一个可部署地块
  - 放置一个 `wallet`
  - 立即结束当前回合
- 钱包规则
  - 只有拥有者自己可以拾取
  - 只有地面移动经过时才触发
  - 触发后移除钱包，并额外投一次工具骰

## 召唤物语义

当前召唤物与地形、Tool 一样，遵守共享层统一结算：

- 地形强调“停留”与“经过”
- 召唤物当前只实现了“经过”
- Tool 是否会触发召唤物，取决于自身的 `passThroughEffectMode`

当前效果：

- `Movement`
  - 会触发钱包拾取
- `Brake`
  - 会触发钱包拾取
- `Jump`
  - 不会触发钱包拾取

## 客户端表现

左栏现在分为三块：

- 角色信息
- 主动技能
- 回合工具

这样角色主动技能不会和普通 Tool 混成一堆，后续如果一个角色拥有多个主动技能，也能保持结构稳定。

场景中则把钱包作为真实的 3D 元素渲染，放置预览直接读取共享层返回的 `summonMutations`。后续新增别的召唤物时，可以复用同一条预览和同步链路。

## 后续扩展建议

如果继续扩展角色系统，优先沿这条路径：

1. 新角色先写进 `characters.ts`
2. 角色能力尽量表达成：
   - 回合赠送 Tool
   - 主动技能 Tool
   - Tool 被动变换
3. 如果能力需要场上实体，再新增召唤物定义与触发规则
4. 客户端只消费同步后的角色快照、Tool 列表和召唤物列表

这样可以尽量避免角色能力演变成前后端双份硬编码。
