

# AI开发规约

1. **文档阅读和维护**
   1. 执行开发任务前，阅读docs/index.md文档确保理解项目架构，按需阅读子文档。
   2. 完成任务后，要持续地维护和跟进文档，并去掉过时内容。
   4. 文档应当包含当前的项目代码架构设计、数据流、核心函数的入口、核心机制的实现。
   5. 文档不应包含过多的内容设计（如每个角色的具体能力），不应包含已修复的bug、过时的架构设计。
   8. 文档始终使用中文编写。
2. **代码规范**
   1. 遵循现有的代码风格和命名规范
   2. 为重要的函数添加注释简要说明其功能，注释只能使用英文ASCLL字符
   3. 不要重复实现一个功能，如果确实需要，请思考是不是架构问题并咨询用户
3. **测试**
   1. 测试时，能正常运行、系统输出值正常即可，不需要进行图像识别测试
   2. 使用npm run goldens进行黄金案例测试
