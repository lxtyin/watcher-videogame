# AI开发规约

## 文档阅读和维护

1. 执行开发任务前，先阅读 `docs/index.md`，按需继续阅读相关架构文档。
2. 完成任务后，必须同步维护文档，并删除过时内容。
3. 文档应当描述当前代码架构、数据流、关键入口文件和核心机制的实现边界。
4. 文档不应堆积角色细节、已修复 bug、废弃设计或迁移过程；这些内容统一沉淀到 `progress.md`。
5. 同一机制只能保留一套权威概念命名。重构后必须同步删除旧术语，避免 `PreviewDescriptor / Presentation / PlaybackEngine` 之类的新旧模型并存。
6. 文档始终使用中文编写。

## 代码规范

1. 遵循现有代码风格和命名规范。
2. 为重要函数添加简短注释说明其职责，注释只能使用英文 ASCII 字符。
3. 不要重复实现同一个功能；如果确实出现重复需求，应先判断是不是架构问题，再决定如何修改。

## 测试

1. 测试以系统能正常运行、关键输出正确为准，不需要依赖图像识别。
2. 优先使用 `npm.cmd run goldens` 进行黄金案例测试。
